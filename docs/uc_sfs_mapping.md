# UC → SFS Mapping Framework

This document maps real-world Universal Credit (UC) concepts to the SunnyBest Forecasting System (SFS).

## Purpose
To simulate real operational, behavioural, and policy-driven challenges from UC within a retail forecasting system.

This allows:
- experimentation in a safe environment
- deeper understanding of forecasting drivers
- transfer of knowledge between public and private sector analytics

---

## 🏛 UC vs 🛒 SFS Mapping

| UC (Work Context) | SFS (Project Context) | Meaning |
|-------------------|-----------------------|---------|
| Claimants | Customers | People receiving value |
| Payments | Sales | Value being delivered |
| Sanctions | Business Rules | Restrictions applied |
| Appointments | Store Activity | Operational interactions |
| Job Centres | Stores | Service points |
| Work Coaches | Store Managers | Decision makers |
| Caseload | Customer Base | Total volume |
| Inflows | New Customers | Entries into the system |
| Conditionality | Promotions / Pricing Rules | Behaviour drivers |
| Fraud & Error | Data Anomalies | Data quality issues |
| Policies | Business Logic | System rules |
| Forecasting UC demand | Sales Forecasting | Core prediction task |

---

## 🧠 How to Use This Framework

For any UC problem:

1. Identify the UC concept  
2. Map it to the SFS equivalent  
3. Simulate the scenario using SFS data  
4. Analyse the impact  
5. Build insights, models, or alerts

To build an AI-powered forecasting system that reflects real-world operational complexity, enabling:
